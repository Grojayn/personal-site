RAG 个人助手

基于检索增强生成技术构建的个人 AI 知识问答系统。喂入个人知识库后，可回答关于技术背景、项目经验、设计理念等问题。

RAG LLM Dify Embedding 向量数据库

项目概述

一个嵌入个人网站的 AI 问答助手，基于 RAG（检索增强生成）技术。将个人背景、技能、项目经验、设计理念等整理为结构化的知识库文本，通过向量嵌入存入数据库。用户提问时，系统检索相关语段作为上下文，再由 LLM 生成准确回答。最终以聊天气泡形式嵌入网站右下角，访问者可以随时对话了解我。

技术链路

知识库构建

将个人简介、技能、项目、设计理念、联系方式整理成分段 TXT，每段标注 #topic 元数据

文本分块

以 \n\n 为分隔符切分知识块，保持语义完整。每块包含足够的上下文信息可独立理解

向量嵌入

将每个文本块通过 Embedding 模型转为高维向量，存入向量数据库，支持语义相似度检索

检索增强

用户提问时，将问题同样向量化，从知识库中检索最相关的 Top-K 语段作为上下文

LLM 生成

将检索到的语段拼接为 System Prompt 喂给 LLM，以第一人称身份回答用户问题

网站嵌入

通过 Dify 的 embed.js 将聊天窗口嵌入个人网站，右下角气泡式交互，支持多轮对话

知识库结构

knowledge-base.txt ├── 个人简介（3段） ├── 技能领域（4段） ├── 技术偏好（2段） ├── 项目经验（7段：网站、ClipboardHistory、PlanClip、游戏） ├── 设计理念（2段） └── 联系方式（1段） 共 19 个知识块，每个 #topic: 标记作为检索元数据
